Blog post upload instructions

You will be adding the blog post to our class website via a pull request on Github. This guide will walk you through the steps. If you help a classmate with their pull request, you can request a regrade on any "Final Choice Exercise" via Gradescope, and I will give you three homework points.

  1. Navigate to the class website repository on Github https://github.com/peterkagey/class-website/.
  2. In the upper right, click "fork", and then on the next page, click "Create fork". This will make a copy of all of the class website content on your account.
  3. On the copy associated with your account, navigate to /content/S26/MAT4170/blog and click Add file > Create new file.
  4. Name your file FirstnameLastname/index.md with your first name and last name (or email me if you use a pseudonym).
  5. Before you add the markdown contents, paste in the following header, appropriately filled out with the blog title, your name, and the date. 
    ---
title: Your Blog Title
author: Your name
date: 2000-01-01
type: blog
courseNumber: MAT4170
term: S26
layout: layouts/blog.njk
---
  6. Under this header, paste in the body of your markdown file. You do not need to include a title, your name, or the date in the body, because it will be automatically populated from the header information you filled out above.
  7. Hit Commit changes... and commit directly to main branch.
  8. Now it's time to add your images or other files so that they're stored locally on the website.
    1. Now there should be a folder called /content/S26/MAT4170/blog/FirstnameLastname. Navigate there, click Add file > Upload files. Upload your files and commit them.
    2. Go fix the references to each of your files in the Markdown source, and commit those changes. (I encourage you to use relative paths, which means that you can refer to these files simply as file.ext instead of as an entire URL.)
    3. (If you're using files besides image files, see the instructions below.)
  9. Now it's time to start your pull request.
    1. Navigate to the "root" of the project at class-website where you should see something that looks like This branch is 2 commits ahead of peterkagey/class-website:main.
    2. Click contribute > Open Pull Request, then type a title and a brief description.
    3. Click Create Pull Request.
    4. I'll get a notification, and when I merge it in and build, your blog post will appear on the website. (Later I'll ask you to check that it looks how you expect, and we can fix any mistakes.)

Making edits

If you ever want to make changes to your blog post, you should just be able to do another pull request. If you are having trouble with this or anything else, reach out for help!

Special file instructions

There are a few other kinds of files you might want to include in your blog post.

GLB files (3D object)

If you want to show a .glb file in your blog post, include the following JavaScript after the header 

<script type="module" src="https://ajax.googleapis.com/ajax/libs/model-viewer/4.0.0/model-viewer.min.js"></script>
Then include the following wherever you want to place the model. 
<model-viewer alt="Your description here." src="filename.glb"></model-viewer>
Make sure to update the alt text and check that the filename matchs what you uploaded.

Video files

While you are welcome include a video as a .mp4 file, it probably will not display very nicely on the website. The easiest thing to do is upload it to Youtube and embed the player. Come chat with me if you don't know how to do this.
